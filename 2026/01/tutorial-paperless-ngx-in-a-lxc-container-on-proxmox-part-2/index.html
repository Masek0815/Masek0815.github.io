Tutorial: Paperless-NGX in a LXC-Container on Proxmox - Part 2

January 12, 2026

Sizing & Preparation

I made these thought

In the last 16 years about 5.000 documents have entered our joint stash with a total volume of about 1GB. Being very generous I decided that about 100 GB should be more the enough to handle expected growth in size and volume of documents.

Without having any experience I decided to allocate 4 virtual CPUs and 16GB of RAM. Other than the disk space, this can later be adjusted easily.

Then there is the quetion of which OS to use as basis. As I am a lazy SOB, I intended to use one of the default offerings from Proxmox. So I logged in via Console to Proxmox and did a

pveam update
pveam available

From the list I decided for “debian-13-standard_13.1-2_amd64.tar.zst” and made that template locally available via

pveam download local debian-13-standard_13.1-2_amd64.tar.zst

Creating the Container

Important note: This is not a Proxmox tutorial. I try to give you the most important steps, but you should roughly understand your Proxmox before this step.

Proxmox has a guided process to setup a container. You select in the WebGUI of Proxmox the “Create CT”-button in the following you go through several steps:

  • Page 1 - General
    • Node: Your Proxmox node on which the LXC container
    • CT ID: An ID for your system, the suggested ID is usually OK
    • Hostname: Which hostname you want to give your instance, in my case “paperless”
    • Resource Pool: Select one of the Pools you created. In my case it was “Prod” for Production. When you setup Proxmox, you should define pools.
    • Password/Confirm Password: The root password
    • SSH public key: SSH-Key (recommended to use this instead of your password)
  • Page 2 - Template
    • Storage: local
    • Template: debian-13-standard_13.1-2_amd64.tar.zst
  • Page 3 - Disks
    • Storage: A storage pool you gave your Proxmox
    • Disk size: (see above)
  • Page 4 - CPU
    • Cores: 4 (see above)
  • Page 5 - Memory
    • Memory: 16384 (see above)
    • Swap: 16384 (see above)
  • Page 6 - Network
    • May be more complicated if you use multiple networks on your Proxmox
    • Select network and gateway (remark: IPv4 CIDR notation is something like 192.168.0.10/24)
  • Page 7 - DNS
    • Hopefully the DNS settings of your Proxmox host are such that you can use those
  • Page 8 - Confirm
    • Check your settings
    • Select “Start after created”
    • Press “Finish” and hope for the best …

VM or Bare Metal

In that case you just install a VM or a bare metal server with Debian 13 and follow the standard setup to set similar parameters as for the LXC container.

uv

Paperless-NGX is written in Python. Managing Python depencies is a mess. Luckily there is a tool for that: https://docs.astral.sh/uv/

Luckily Paperless-NGX is prepared to run with uv. So I make use of this.

Remark: “uv” is worthy of a tutorial on itself. @kris has started one here: https://github.com/isotopp/uv-class. In short: it allows you to specify all Python depencies of your in a single file (“uv.lock”) and sync your local installation to it with a single command (“uv sync”).

OS preparation

Either via SSH or Console, you login as “root” to your system.

First make sure all your packages are up to date:

apt-get update
apt-get upgrade

Then you need to install a lot of packages:

apt-get install imagemagick fonts-liberation gnupg libpq-dev default-libmysqlclient-dev pkg-config libmagic-dev libzbar0 poppler-utils git unpaper ghostscript icc-profiles-free qpdf libxml2 pngquant zlib1g tesseract-ocr libxslt1-dev media-types mailcap  build-essential redis curl tesseract-ocr-deu sudo

Then you fix your locale:

export LANGUAGE=en_US.UTF-8
export LANG=en_US.UTF-8
export LC_ALL=en_US.UTF-8
locale-gen en_US.UTF-8
dpkg-reconfigure locales

If you’re German like me, you will rather use “de_DEU.UTF-8” instead of “en_US.UTF-8”.

Next you create the “paperless” user on the system:

adduser paperless --system --home /opt/paperless --group
usermod -s /bin/bash paperless

Since I need to do some ops as user “paperless”, I give it a shell. That can be removed later.

Then I switch to over to the user “paperless” with

su - paperless

Software Installation

As user “paperless” I execute the next steps:

Install “uv”:

curl -LsSf https://astral.sh/uv/install.sh | sh
source $HOME/.local/bin/env

Download and unpack the software

wget https://github.com/paperless-ngx/paperless-ngx/releases/download/v2.20.0/paperless-ngx-v2.20.0.tar.xz
tar xvf paperless-ngx-v2.20.0.tar.xz
rm paperless-ngx-v2.20.0.tar.xz

Remark: this is version v2.20.0. You might want to use a more recent version when you come up on this documentation there will be a newer one. See https://github.com/paperless-ngx/paperless-ngx/releases for the most current release.

Now move it to the right place:

cd paperless-ngx
mv * ..
mv .* ..
cd ..
rmdir paperless-ngx

Now sync your Python:

uv sync --extra granian

The “–extra” installs a module that is IMHO missing from the “uv.lock”

We wish Paperless-NGX to be able to process natural language. Luckily there is a package (“ntlk”) for that too:

uv add nltk
uv run python -m nltk.downloader stopwords
uv run python -m nltk.downloader punkt
uv run python -m nltk.downloader punkt_tab

These are packages recommended for the German language. I would ask someone more familiar with “ntlk” to pitch in, what you should add for other languages.

Now create some directories that Paperless-NGX needs:

mkdir /opt/paperless/media /opt/paperless/data /opt/paperless/consume

Next you need to edit the config. See the next chapter for a barebone config:

vi paperless.conf

The next step is running the initiation scripts:

cd src
uv run manage.py migrate
uv run manage.py collectstatic

Then you’re done with the “uv” user an can exit the shell:

exit

Sample Config

What do you need to edit?

  • Your secret: in my example it is “topsecret”
  • Your FQDN for the server: in my example it is “paperless.example.org”
  • Your OCR language (for analyzing text, dates, numbers): in my example it is “deu” for German
  • Your time zone: in my example it is Europe/Berlin

There are a lot more settings that you can use. See https://docs.paperless-ngx.com/configuration/ for more details.

PAPERLESS_REDIS=redis://localhost:6379
PAPERLESS_CONSUMPTION_DIR=/opt/paperless/consume
PAPERLESS_DATA_DIR=/opt/paperless/data
PAPERLESS_MEDIA_ROOT=/opt/paperless/media
PAPERLESS_STATICDIR=/opt/paperless/static
PAPERLESS_SECRET_KEY=topsecret
PAPERLESS_URL=http://paperless.example.org:8000
PAPERLESS_CSRF_TRUSTED_ORIGINS=http://paperless.example.org,https://paperless.example.org
PAPERLESS_ALLOWED_HOSTS=paperless.example.org
PAPERLESS_OCR_LANGUAGE=deu
PAPERLESS_TIME_ZONE='Europe/Berlin'

Startup scripts

Now continue as root user to make the system start properly. First copy the startup scripts

cp /opt/paperless/scripts/paperless-webserver.socket /etc/systemd/system
cp /opt/paperless/scripts/paperless-webserver.service /etc/systemd/system
cp /opt/paperless/scripts/paperless-consumer.service /etc/systemd/system
cp /opt/paperless/scripts/paperless-task-queue.service /etc/systemd/system
cp /opt/paperless/scripts/paperless-scheduler.service /etc/systemd/system

Then edit them:

vi /etc/systemd/system/paperless-webserver.service
vi /etc/systemd/system/paperless-consumer.service
vi /etc/systemd/system/paperless-scheduler.service
vi /etc/systemd/system/paperless-task-queue.service

The following changes need to be made:

In “/etc/systemd/system/paperless-webserver.service”

Instead of:

exec granian --interface asginl --ws "paperless.asgi:application"'

you use:

exec /opt/paperless/.local/bin/uv run granian --interface asginl --ws "paperless.asgi:application"'

In /etc/systemd/system/paperless-consumer.service

Instead of:

ExecStart=python3 manage.py document_consumer

you use:

ExecStart=/opt/paperless/.local/bin/uv run manage.py document_consumer

In “"/etc/systemd/system/paperless-scheduler.service” and “paperless-task-queue.service”

Instead of:

ExecStart=celery --app paperless beat --loglevel INFO

you use:

ExecStart=/opt/paperless/.local/bin/uv run celery --app paperless beat --loglevel INFO

Then you can enable those services:

systemctl enable paperless-webserver.socket
systemctl enable paperless-webserver
systemctl enable paperless-consumer.service
systemctl enable paperless-scheduler.service
systemctl enable paperless-task-queue.service

and then start them:

systemctl start paperless-webserver.socket
systemctl start paperless-webserver.service
systemctl start paperless-consumer.service
systemctl start paperless-scheduler.service
systemctl start paperless-task-queue.service

Please remember to check on those services in order to see if there is any problem:

systemctl status paperless-webserver.socket
systemctl status paperless-webserver.service
systemctl status paperless-consumer.service
systemctl status paperless-scheduler.service
systemctl status paperless-task-queue.service

Best do this service by service.
