Day 107: Blogging with Hugo (Part 1)

July 5, 2020

This article is part of The 100 Days Offensive. Go to Day 102

As already mentioned, I decided to migrate to Hugo as blog software. Furthermore now GitHub is hosting my pages instead of Google. Though the first steps were rather simple, my specific requirements created some challenges.

Requirements

With my new blog I wanted to achieve several things:

  1. I wanted to leave Google and the Blogger platform behind. Initially I already to be free of Google at the end of last year, but the process takes longer than expected. I guess next week I will start to migrate away from Google Email.
  2. Another goal was to publish my blog as a collection of static HTML files. From the security perspective, this simplifies the challenge a lot. Database and Scripting engines require permanent updates and (as more complex tools) offer a lot of opportunity to do things wrong.
  3. If possible, I wanted to write my Blog in Markdown and with my (current favourite) editor Typora.
  4. I would prefer the source files to be on a local directory or a share of my Windows 10 PC.
  5. The URLs of the original blog must continue to work after the migration.

System Setup

In order to use Hugo on my Windows PC I could have used the Windows binary. But since I wanted to be able to script some processing (which I knew would happen at some point), I wanted to have the standard Unix tools (find, grep, sed) available. So the decision was to use the Windows Subsystem for Linux. I used that as an excuse to switch to the Version 2.

Little did I know that I created the first hurdle. As it turned out, I could install version 2 quite easily. But the upgrade with wsl --set-version Ubuntu 2 would always give me the error message “Please enable the Virtual Machine Platform Windows feature and ensure virtualization is enabled in the BIOS."

As I was already running VMware Workstation on my PC, I was completely sure that the virtualization was already enabled in the BIOS. As it turned out, this was not the case. It took me more than one hour to finally check the BIOS and it was not enabled. Unluckily entering the BIOS is quite a hassle for me as the PC and the monitor/keyboard are quite a way apart. So I use an USB extender that takes some times to initialize and often the boot sequence is already beyond the “Press DEL to enter BIOS”-phase. Luckily the “Method #2: Use Windows 10’s start menu” worked for me.

As it turned out, that was not enough. I also needed to update my VMware Workstation to a new version (AFAIK at least version 15.5 is required).

After that, the update to WSL2 worked and I could start my Linux shell. Recommendation at this point, use the new Windows Terminal instead of the default terminal. That gives you the ability to use tabs to separate multiple shells.

The installation of Hugo itself on WSL2 is quite easy: sudo apt-install hugo

Preparing the environment

First you have to decide where you want to have your site. In my case I created a Folder called “Hugo” in the “Documents”-folder of my local Windows user.

When you start your shell, this translates to /mnt/c/Users/myuser/Documents/Hugo, where “myuser” is your local Windows user name. On the shell I switch into that directory and execute the following commands:

hugo new site Demo

This creates a subdirectory “Demo” where your new site resided. Change into that subdirectory. For further reference I will call this directory now “BASEDIR”.

Now it is time to get your first theme (visual template). I used the recommended Ananke Theme, which is a sensible choice. To install it used the commands:

git init

git submodule add https://github.com/budparr/gohugo-theme-ananke.git themes/ananke

You find the files in “BASEDIR/themes/ananke” and there is a “README.md” file in it for customization purposes.

In order to create the content of your first post, in the BASEDIR you execute the command

hugo new posts/my-first-post.md

All the content goes to “BASEDIR/content”, so your post ends up in “BASEDIR/content/posts/my-first-post.md”. Edit the file and make it look like:

---
title: "My First Post"
date: 2020-07-05T08:47:11+01:00
draft: true
---

Hello

Back in the BASEDIR, now execute command:

hugo server -D

This starts a small web server at http://localhost:1313/ (if you did not change the default settings) where you can see your first site. Stopping the Server with Ctrl-C and starting it again rebuilds your site if you changed or added anything.

In order to create more content, you just add files to the directory ““BASEDIR/content/posts”. I added that directory to the quick access list of the Windows Explorer. But (by default) Windows does not allow to create new Markdown files though the context menu. But since that would simplify my content pipeline, I investigated on how to create such an option.

Step 1: Create the file “c:\WINDOWS\ShellNew\template.md” (you may need to create the directory as well, both requires admin privileges with Windows 10).

Step 2: Put the following content in your Markdown-Template (you may change this to fit your needs):

---
title: ""
date: 2020-06-25T12:00:00+02:00
summary: ""
draft: true
disable_share: true
aliases:
  - ""

---

**Summary**

Text

In this template I already used some features. The “title” has to contain the headline of your post. Into “summary” I put the short text that is shown on the main page for the article. With “alias” I can specify an additional URL (e.g. “/about.html”) that can be used to reach this article. Important: When you create your files from a template, you have to adjust “date” by hand. Otherwise all your posts are shown with the date from your template file. I’m looking for a method to fix that.

Step 3: Create a short text file with the following content:

### Windows Registry Editor Version 5.00

[HKEY_CLASSES_ROOT\.md]
@="markdown"

[HKEY_CLASSES_ROOT\.md\ShellNew]
"NullFile"=""
"FileName"="C:\Windows\ShellNew\template.md"

[HKEY_CLASSES_ROOT\markdown]
@="New Blog Post"

And import that file into the Windows Registry Editor (regedit.exe)

After a reboot you should now have “New Blog Post” as an option when you select “New” from the context menu in the Windows Explorer and you have finished the basic setup. In the next post I will explain the site parameters and the deployment.
